1-3) Computes the value of base raised to the power exponent.

4) Type-generic macro: If any argument has type long double, `powl` is called. Otherwise, if any argument has integer type or has type double, `pow` is called. Otherwise, `powf` is called. ### [[edit](https://en.cppreference.com/mwiki/index.php?title=c/numeric/math/pow&action=edit&section=2 "Edit section: Return value")] Return value

If no errors occur, base raised to the power of exponent (baseexponent  
) is returned. 

If a domain error occurs, an implementation-defined value is returned (NaN where supported). 

If a pole error or a range error due to overflow occurs, ±[HUGE_VAL](HUGE_VAL.html), `±HUGE_VALF`, or `±HUGE_VALL` is returned. 

If a range error occurs due to underflow, the correct result (after rounding) is returned. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=c/numeric/math/pow&action=edit&section=3 "Edit section: Error handling")] Error handling

Errors are reported as specified in [`math_errhandling`](math_errhandling.html "c/numeric/math/math errhandling"). 

If base is finite and negative and exponent is finite and non-integer, a domain error occurs and a range error may occur. 

If base is zero and exponent is zero, a domain error may occur. 

If base is zero and exponent is negative, a domain error or a pole error may occur. 

If the implementation supports IEEE floating-point arithmetic (IEC 60559), 

  * pow(+0, exponent), where exponent is a negative odd integer, returns `+∞` and raises [FE_DIVBYZERO](../fenv/FE_exceptions.html "c/numeric/fenv/FE exceptions")
  * pow(-0, exponent), where exponent is a negative odd integer, returns `-∞` and raises [FE_DIVBYZERO](../fenv/FE_exceptions.html "c/numeric/fenv/FE exceptions")
  * pow(±0, exponent), where exponent is negative, finite, and is an even integer or a non-integer, returns +∞ and raises [FE_DIVBYZERO](../fenv/FE_exceptions.html "c/numeric/fenv/FE exceptions")
  * pow(±0, -∞) returns +∞ and may raise [FE_DIVBYZERO](../fenv/FE_exceptions.html "c/numeric/fenv/FE exceptions")(until C23)
  * pow(+0, exponent), where exponent is a positive odd integer, returns +0 
  * pow(-0, exponent), where exponent is a positive odd integer, returns -0 
  * pow(±0, exponent), where exponent is positive non-integer or a positive even integer, returns +0 
  * pow(-1, ±∞) returns 1
  * pow(+1, exponent) returns 1 for any exponent, even when exponent is `NaN`
  * pow(base, ±0) returns 1 for any base, even when base is `NaN`
  * pow(base, exponent) returns `NaN` and raises [FE_INVALID](../fenv/FE_exceptions.html "c/numeric/fenv/FE exceptions") if base is finite and negative and exponent is finite and non-integer. 
  * pow(base, -∞) returns +∞ for any `|base|<1`
  * pow(base, -∞) returns +0 for any `|base|>1`
  * pow(base, +∞) returns +0 for any `|base|<1`
  * pow(base, +∞) returns +∞ for any `|base|>1`
  * pow(-∞, exponent) returns -0 if exponent is a negative odd integer 
  * pow(-∞, exponent) returns +0 if exponent is a negative non-integer or negative even integer 
  * pow(-∞, exponent) returns -∞ if exponent is a positive odd integer 
  * pow(-∞, exponent) returns +∞ if exponent is a positive non-integer or positive even integer 
  * pow(+∞, exponent) returns +0 for any negative exponent
  * pow(+∞, exponent) returns +∞ for any positive exponent
  * except where specified above, if any argument is NaN, NaN is returned. 



### [[edit](https://en.cppreference.com/mwiki/index.php?title=c/numeric/math/pow&action=edit&section=4 "Edit section: Notes")] Notes

Although `pow` cannot be used to obtain a root of a negative number, [cbrt](cbrt.html "c/numeric/math/cbrt") is provided for the common case where `exponent` is 1 / 3. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=c/numeric/math/pow&action=edit&section=5 "Edit section: Example")] Example

Run this code
    
    
    #include <errno.h>
    #include <fenv.h>
    #include <math.h>
    #include <stdio.h>
    // #pragma STDC FENV_ACCESS ON
     
    int main(void)
    {

        [printf](../../io/fprintf.html)("pow(2, 0.5) = %f\n", pow(2, 0.5));
        [printf](../../io/fprintf.html)("pow(-2, -3) = %f\n", pow(-2, -3));
     
        // special values
        [printf](../../io/fprintf.html)("pow(-1, NAN) = %f\n", pow(-1, NAN));
        [printf](../../io/fprintf.html)("pow(+1, NAN) = %f\n", pow(+1, NAN));
        [printf](../../io/fprintf.html)("pow(INFINITY, 2) = %f\n", pow(INFINITY, 2));
        [printf](../../io/fprintf.html)("pow(INFINITY, -1) = %f\n", pow(INFINITY, -1));
     
        // error handling
        errno = 0; [feclearexcept](../fenv/feclearexcept.html)([FE_ALL_EXCEPT](../fenv/FE_exceptions.html));
        [printf](../../io/fprintf.html)("pow(-1, 1/3) = %f\n", pow(-1, 1.0 / 3));
        if (errno == [EDOM](../../error/errno_macros.html))
            [perror](../../io/perror.html)("    errno == EDOM");
        if ([fetestexcept](../fenv/fetestexcept.html)([FE_INVALID](../fenv/FE_exceptions.html)))

    }
